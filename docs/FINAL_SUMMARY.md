## 🚀 使用指南

### 1. 启动服务
```bash
./run.sh
```

### 2. 配置 LLM（启用翻译）

1. 打开 http://127.0.0.1:8000
2. 滚动到 **⚙️ LLM Configuration**
3. 填写配置：
   - Provider: 选择 `OpenAI-Compatible` 或 `Anthropic (Claude)`
   - Base URL: 填写 API 地址（如 `https://api.openai.com/v1`）
   - API Key: 填写你的 API Key
4. 点击 **"Test Connection & Get Models"**
5. 等待系统获取可用模型列表
6. 从列表中**点击选择**一个模型（会高亮显示）
7. 点击 **"Save Configuration"**
8. 重启服务：`./run.sh`

### 3. 上传文档并自动翻译

1. 在 **Upload manual** 区域选择 PDF 或 TXT 文件
2. 可选：填写产品标题
3. 点击 **Upload**
4. 系统自动：
   - 检测源语言
   - 翻译成 7 种语言（需要配置 LLM）
   - 索引到向量库
   - 同步到文档中心

### 4. 批量下载 PDF

1. 滚动到 **📚 Document Center**
2. 在 **Select Languages** 区域勾选想要的语言
3. 在表格中勾选想要下载的文档
4. 点击 **"Download Selected PDFs"**
5. 浏览器自动下载所有 PDF

## 📊 API 端点

### LLM 配置相关
- `GET /api/llm/config` — 获取当前配置
- `POST /api/llm/config` — 保存配置到 .env
- `POST /api/llm/test-connection` — 测试连接并获取模型列表（新增）

### 文档管理
- `GET /api/documents` — 获取所有文档
- `POST /api/manual/upload` — 上传文档
- `POST /api/manual/{id}/pdf` — 生成 PDF
- `DELETE /api/manual/{id}` — 删除文档

### 问答
- `POST /api/ask` — 提问

### 健康检查
- `GET /health` — 服务状态

## 🎓 设计原则

### 1. 渐进式披露
- 默认只显示必要信息
- 点击"测试连接"后才显示模型列表
- 选择模型后才显示"保存"按钮

### 2. 即时反馈
- 每个操作都有状态提示
- 成功/失败有明确的视觉反馈
- 加载状态清晰可见

### 3. 防错设计
- 必填字段有明确标识
- 危险操作（删除）需要确认
- 内置产品不可删除

### 4. 一致性
- 统一的间距系统
- 统一的颜色系统
- 统一的交互模式
